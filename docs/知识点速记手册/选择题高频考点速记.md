## 一、机器学习基础（必考3-5题）

### 1. 过拟合 vs 欠拟合
- **过拟合**: 训练集准确率高，测试集低 → 模型太复杂/数据太少 → 加正则化/Dropout/早停/数据增强
- **欠拟合**: 训练集和测试集都低 → 模型太简单 → 增加复杂度/更多特征

### 2. 正则化
| 类型 | 公式 | 特点 |
|-----|------|------|
| L1(Lasso) | λΣ|w| | 产生稀疏解(特征选择) |
| L2(Ridge) | λΣw² | 权重趋近0但不为0 |

### 3. 降维算法
- **PCA**: 最大化方差，最小化重构误差，基于特征值分解/协方差矩阵
- **NMF**: 数据必须非负，分解后具有可解释性（"部分的叠加"），常用于文本聚类/图像特征提取

### 4. 常见损失函数
| 任务 | 损失函数 | 说明 |
|-----|---------|------|
| 二分类 | BCEWithLogitsLoss | 内置sigmoid |
| 多分类 | CrossEntropyLoss | 内置softmax |
| 回归 | MSELoss / MAELoss | 平方/绝对值 |

### 5. 激活函数
- **Sigmoid**: 输出(0,1)，梯度消失，非零中心
- **Tanh**: (-1,1)，零中心，仍可能梯度消失
- **ReLU**: max(0,x)，计算快，缓解梯度消失；**Dead ReLU**(负半轴梯度为0)
- **LeakyReLU**: 解决Dead ReLU，负半轴斜率0.01
- **GELU**: Bert/GPT使用，比ReLU平滑

### 6. 混淆矩阵与F1
```
           预测正     预测负
实际正      TP         FN
实际负      FP         TN
```
- Precision = TP/(TP+FP) — 预测为正的里面多少正确，准确率(accuracy)，精确率(precision)
- Recall = TP/(TP+FN) — 真正例被找出的比例
- F1 = 2×P×R/(P+R) — 调和平均

### 7. 优化器
- **SGD+Momentum**: 累计动量加速收敛
- **Adam**: 自适应学习率，最常用
- **AdamW**: Adam + 解耦权重衰减(推荐)

### 8. 特征标准化
- StandardScaler: (x-μ)/σ → 均值0方差1
- MinMaxScaler: (x-min)/(max-min) → [0,1]
- Var(aX+b) = a²·Var(X) ★高频计算题

### 9. EM算法、LDA主题模型的理解

## 二、深度学习（必考3-5题）

### 1. LSTM
- 解决RNN长序列梯度消失问题
- **三"门"一状态**: 遗忘门、输入门、输出门、细胞状态
- 适合文本生成、时间序列预测

### 2. Transformer
- **核心**: Self-Attention: Attention(Q,K,V) = softmax(QK^T/√d_k)V
- **多头注意力**: 多组QKV并行计算后拼接，h个头输出拼接维度还是D
- **Position Encoding**: 正余弦/可学习/RoPE
- **Feed Forward**: 两层线性+激活，输入输出维度不变(D→4D→D)
- **LayerNorm + 残差连接**: 每个子层后都有

### 3. BatchNorm vs LayerNorm
- **BN**: 对batch维度标准化 → 受batch size影响，训练用batch统计，推理用滑动平均
- **LN**: 对每个样本的特征维度标准化 → 不受batch size影响，Transformer使用

### 4. Dropout
- 训练时随机丢弃p，推理时乘以(1-p)保持期望一致
- 作用: 防过拟合，隐式集成

### 5. GQA (Grouped Query Attention)
- 多个Query头共享一组Key/Value
- 例: h=8头，GQA分组g=2 → 每2个头共享K/V，共4组
- 作用: 减少KV-cache，加速推理

### 6. MobileNet
- 轻量级网络，使用深度可分离卷积（Depthwise Separable Convolution）
- 大幅减少参数量和计算量

### 7. FP16混合精度
- 用半精度(FP16)训练，用FP32做梯度累积
- 好处: 减少显存占用，加速计算

### 8. 多任务学习
- 共享底层表示，同时学习多个相关任务
- 提高泛化能力，减少过拟合

## 三、大模型专项（必考3-5题）

### 1. RLHF (Reinforcement Learning from Human Feedback)
- **三阶段**: SFT微调 → 训练奖励模型(RM) → PPO强化学习
- **奖励模型训练依赖**: 人类偏好排序数据 ★常考
- **PPO-KL惩罚**: 防止策略偏离参考模型过远，用 KL(π_θ || π_ref) 正则项
- **RLHF问题**: 奖励黑客、模式崩溃、分布外退化

### 2. MoE (Mixture of Experts)
- **门控网络**: 为每个token选择Top-k个专家
- **负载均衡**: 防止门控坍缩到少数专家，用负载均衡损失系数
- **Top-2 + 容量因子1.0**: 每个token最多被2个专家处理
- **训练时**: 门控网络参数会更新(不冻结) ★常见错误选项

### 3. FlashAttention
- **核心思想**: 利用SRAM(片上高速缓存)，减少HBM(显存)访问次数
- **内存复杂度**: O(n²) → O(n) ★常考
- **数值精度**: 与标准Attention数学上完全等价
- **通过分块tiling + 重计算实现**

### 4. 模型量化
- INT4量化: FP32的32bit → 4bit, 存储缩小约87.5%(≈1/8)
- PTQ(训练后量化): LLM.int8(), SmoothQuant, GPTQ
- QLoRA: 量化感知微调(不是纯PTQ)

### 5. RAG (Retrieval Augmented Generation)
- 先检索外部知识库，再让大模型基于检索结果生成
- 最直接缓解"幻觉"的方法 ★常考

### 6. RoPE位置编码
- 旋转位置编码，可通过基频缩放(base rescaling)微调后外推更长上下文

### 7. PagedAttention
- 将KV-cache分页管理，解决显存碎片化问题
- vLLM的核心技术

### 8. 张量并行
- 将模型参数切分到多张GPU上并行计算

### 9. ZeRO-3
- 优化器状态、梯度、参数均分片到各卡
- 64卡训练，每卡显存约为单卡的1/64

### 10. Constitutional AI
- 模型自写批评规则集，自我审查修正输出

### 11. 涌现能力
- 模型参数在100亿到1000亿区间更可能产生涌现能力

### 12. 自注意力计算量
- 与序列长度平方成正比: O(n²)
- 提示长度从1k→8k: 计算量放大64倍

## 四、数学基础（必考3-5题）

### 1. 线性代数
- **线性变换**: T(αu+βv) = αT(u)+βT(v)
- 零向量映射到零向量
- 矩阵乘法: C_ij = Σ A_ik × B_kj
- 实对称正定矩阵 → Cholesky分解存在且唯一 A=LL^T
- 病态问题: 系数矩阵行列式接近0

### 2. 概率统计
- 均匀分布: f(x)=1/(b-a), a≤x≤b
- 独立事件: P(A∩B) = P(A)×P(B)
- 方差性质: Var(aX+b) = a²Var(X) ★必考
- 无偏估计量更有效: 方差更小 D(θ̂₁) < D(θ̂₂)

### 3. 数值计算
- 牛顿迭代法: x_{n+1} = x_n - f(x_n)/f'(x_n)
- 局部收敛性（需初始值接近真根）
- 插值误差计算

| 高频考点 | 正确答案特征 | 易错项 |

| 文本生成模型选哪个？ | LSTM | HMM(序列标注)、决策树(分类) |
