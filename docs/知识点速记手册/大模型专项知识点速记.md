# 华为AI岗 - 大模型专项知识点速记

---

## 1. Transformer架构（绝对核心）

```
输入 → [Token Embedding + Position Encoding] → Multi-Head Self-Attention → Add&LN → FFN → Add&LN → 输出
```

**关键公式**：
- Attention(Q,K,V) = softmax(Q·K^T / √d_k) · V
- Q=linear_q(X), K=linear_k(X), V=linear_v(X)
- 除以√d_k：防止内积过大导致softmax饱和

**多头注意力**：
- h个头，每头维度 d_h = D/h
- 各头独立计算后拼接 → Linear(concat) → 输出维度还是D
- 优势：捕捉不同子空间的信息

**FFN**：两层全连接 D→4D→D，中间一般用ReLU/GELU

## 2. RLHF 三阶段

```
① SFT微调 → ② 训练Reward Model → ③ PPO强化学习
```

| 阶段 | 做什么 | 关键点 |
|-----|-------|-------|
| SFT | 用人工标注数据微调预训练模型 | 有监督学习 |
| RM训练 | 训练奖励模型预测人类偏好 | **需要人类偏好排序数据** ★必考 |
| PPO | 用RM的奖励优化策略模型 | 加KL惩罚防偏离 |

**RLHF常见问题**：奖励黑客(找到漏洞刷高分)、模式崩溃(输出单一)、分布外退化

## 3. MoE (Mixture of Experts)

**结构**：门控网络 + 多个Expert(每个是FFN)

**关键点**：
- Top-2门控：每个token选择得分最高的2个专家
- 容量因子1.0：token不复制，最多被2个专家处理
- 门控网络参数也会训练（不冻结）★常考
- 负载均衡损失：防止门控总选少数专家

## 4. 位置编码

| 类型 | 特点 |
|-----|------|
| Sinusoidal(正余弦) | 不需要学习，可外推 |
| 可学习位置编码 | 需要训练，受训练长度限制 |
| RoPE(旋转位置编码) | 通过旋转矩阵编码相对位置，可通过**基频缩放**微调后外推 |

---

## 5. FlashAttention

- **原理**：分块tiling + 重计算，利用SRAM减少HBM访问
- **内存复杂度**：O(n²) → **O(n)** ★必考
- **数值精度**：与标准Attention**数学等价**（只是计算顺序不同）

---

## 6. 模型量化

| 类型 | 方法 | 说明 |
|-----|------|------|
| PTQ(训练后量化) | LLM.int8(), SmoothQuant, GPTQ | 不需要训练 |
| QAT(量化感知训练) | QLoRA | 需反向传播 |

INT4量化：FP32的32bit→4bit，存储缩小至1/8≈87.5%

---

## 7. GQA (Grouped Query Attention)

- 多个Query头共享Key/Value
- h=8, g=2 → 每2个头共享K/V，共4组
- 优势：减少KV-cache，加速推理

---

## 8. PagedAttention

- KV-cache分页管理，解决显存碎片化
- vLLM框架核心

---

## 9. 并行策略

| 策略 | 说明 |
|-----|------|
| 数据并行 | 各卡一份完整模型，数据分片 |
| 张量并行 | 一层内参数切分到多卡 |
| 流水线并行 | 不同层放到不同卡 |
| ZeRO-3 | 优化器+梯度+参数全分片，64卡=1/64显存/卡 |

---

## 10. 数据中心的关键组件

**Attention is All You Need** 中的结构：
- **Encoder**: 6层，每层 = MHA + FFN + Add&LN
- **Decoder**: 6层，每层 = Masked MHA + Cross-Attention + FFN + Add&LN
- Decoder的Cross-Attention：Q来自解码器，K,V来自编码器输出
