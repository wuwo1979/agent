# 错误码对照表

MCP Agent Gateway v2.0 使用 **JSON-RPC 2.0 标准错误码** + **项目自定义扩展错误码**，所有错误通过 `{"jsonrpc":"2.0","error":{"code":N,"message":"..."}}` 格式返回。

---

## 一、JSON-RPC 2.0 标准错误码

| 错误码 | 名称 | 含义 | 触发条件 |
|--------|------|------|---------|
| **-32700** | Parse Error | JSON 语法解析失败 | 请求体不是合法 JSON |
| **-32600** | Invalid Request | 请求结构不符合 JSON-RPC 规范 | 缺少 `jsonrpc`、`method` 必要字段 |
| **-32601** | Method Not Found | 请求的方法不存在 | `method` 值不在 `tools/list`、`tools/call`、`initialize` 中 |
| **-32603** | Internal Error | 服务端内部错误 | 工具执行时抛出未预期的异常 |

## 二、项目自定义错误码

| 错误码 | 名称 | 含义 | HTTP 状态码 | 触发条件 |
|--------|------|------|------------|---------|
| **-32001** | Permission Denied | 权限/认证不足 | 401 / 403 | API Key 无效、工具不在租户白名单、路径越权 |
| **-32002** | Rate Limited | 请求频率超限 | 429 | 超过 `security.rate_limit.requests_per_minute` 阈值 |
| **-32003** | Backend Unavailable | 后端服务不可用 | 502 | Ollama 服务未启动或已崩溃（故障隔离返回） |
| **-32004** | Validation Error | 工具参数校验失败 | 400 | 必填参数缺失、参数类型错误、字符串超长 |
| **-32005** | Timeout | 工具执行超时 | 504 | 超过 `tools.terminal.command_timeout_seconds` 等超时配置 |
| **-32006** | Resource Exceeded | 资源限制超限 | 413 | 文件超过 `workspace.max_file_size_kb`、响应体超限 |
| **-32099** | Gateway Error | 网关内部异常 | 500 | 配置错误、模块依赖缺失等内部故障 |

## 三、错误响应示例

### 3.1 API Key 无效
```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "error": {
    "code": -32001,
    "message": "Permission denied: invalid API key"
  }
}
```
**排查：** 检查请求头 `X-API-Key` 是否在 `config.yaml` 的 `security.auth.api_keys` 列表中。

### 3.2 Ollama 服务不可用（故障隔离）
```json
{
  "jsonrpc": "2.0",
  "id": "2",
  "error": {
    "code": -32003,
    "message": "Backend unavailable: Ollama service is not reachable at http://localhost:11434"
  }
}
```
**排查：** 确认 Ollama 进程是否正在运行，`ollama list` 是否正常返回。
**特性：** 此错误不会影响其他工具（文件读写、命令执行等）的正常使用。

### 3.3 文件路径越权
```json
{
  "jsonrpc": "2.0",
  "id": "3",
  "error": {
    "code": -32001,
    "message": "Permission denied: path /etc/passwd is outside allowed directories"
  }
}
```
**排查：** 检查路径是否在 `workspace.allowed_dirs` 白名单内。

### 3.4 请求频率超限
```json
{
  "jsonrpc": "2.0",
  "id": "4",
  "error": {
    "code": -32002,
    "message": "Rate limit exceeded: 60 requests per minute"
  }
}
```
**排查：** 降低请求频率，或调整 `security.rate_limit.requests_per_minute` 配置。

## 四、HTTP 状态码映射（REST 模式）

| JSON-RPC 错误码 | HTTP 状态码 | 说明 |
|-----------------|------------|------|
| -32700, -32600 | 400 Bad Request | 请求格式错误 |
| -32601 | 404 Not Found | 方法不存在 |
| -32004 | 400 Bad Request | 参数校验失败 |
| -32001 | 401 / 403 Forbidden | 权限/认证问题 |
| -32002 | 429 Too Many Requests | 频率限制 |
| -32003 | 502 Bad Gateway | 后端不可用 |
| -32005 | 504 Gateway Timeout | 执行超时 |
| -32006 | 413 Payload Too Large | 资源超限 |
| -32603, -32099 | 500 Internal Server Error | 内部错误 |

## 五、错误排查速查表

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| `-32001` 认证失败 | API Key 错误或缺失 | 确认请求头 `X-API-Key` 值正确 |
| `-32001` 路径越权 | 文件不在白名单 | 检查 `workspace.allowed_dirs` 配置 |
| `-32002` 频率限制 | 请求过快 | 降低频率或调整 `rate_limit` 配置 |
| `-32003` 后端不可用 | Ollama 未启动 | `ollama serve` 启动服务后重试 |
| `-32004` 参数错误 | 必填参数缺失 | 检查工具参数 Schema |
| `-32005` 超时 | 命令/请求时间过长 | 适当增加超时配置 |
| `-32700` 语法错误 | JSON 格式错误 | 验证请求体 JSON 语法 |
