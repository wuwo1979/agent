## 核心数据流

```
1. 用户输入 → Planner Agent（拆解为子任务 DAG）
2. 子任务 DAG → Executor Agent（依赖分析 + 并行调度）
3. 工具调用 → MCP Gateway（协议路由 + 执行）
4. 工具结果 → 增量缓存（去重压缩）
5. 执行结果 → Validator Agent（质量校验）
6. 校验通过 → 返回最终结果
   校验失败 → 返回 Planner 重新规划（最多 3 次）
```

## 核心指标

| 指标 | 数值 | 说明 |
|------|------|------|
| 内置工具数 | 14 个 | 文件系统(5) + 终端(3) + 数据库(5) |
| Token 压缩率 | 49.5% | 增量上下文缓存 |
| 并行加速比 | 2.9x | 6 工具并行 vs 串行 |
| 耗时降低 | 65.0% | 无依赖工具并行调度 |
| Agent 重试次数 | 3 次 | 指数退避 + 熔断保护 |
| Docker 服务数 | 7 个 | MCP 网关 + Ollama + ChromaDB + Milvus |

## 快速启动

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动 MCP 网关
python main.py --host 0.0.0.0 --port 9090

# 3. 运行演示
python main.py --demo

# 4. 运行跑分
python main.py --benchmark

# 5. 运行测试
python main.py --test

# 6. Docker 一键部署
cd docker && docker-compose up -d
```

## 技术栈

- **协议**: MCP (Model Context Protocol) + JSON-RPC 2.0
- **编排**: LangGraph (StateGraph + Checkpoint)
- **LLM**: DeepSeek-V4 API / Ollama (Qwen2.5, Llama3.1) / vLLM
- **向量库**: ChromaDB / Milvus
- **部署**: Docker Compose (7 服务编排)
- **测试**: pytest + asyncio
- **语言**: Python 3.11+ (全异步)
